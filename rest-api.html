Generic Rest API

2934 views May 14, 2018 laravel-bap 5

Generic rest API allows managing application entities. The user must be logged in via JWT Authentication. API checks if the user has access to single records and list of records base on user roles and permissions.

Generic rest API is a module added in CRM application – but it can be easily transferred to other applications.
Currently, Generic rest API is implemented for  Leads, Contacts, and Tickets.

Login to API as user

POST|GET on {URL}/api/login

params:
email - user email
password - user password

result:
{
    "status": true,
    "data": {
        "access_token": ".....",
        "user": {
            "id": 1,
             ......
        }
    },
    "message": [],
    "errors": []
}

Generic Rest API Methods

Example for leads.

Important: All methods require JWT Authentication. You need to send in a header with values:

Authorization : "Bearer {accesss_token}"
Content-Type  : "application/json"
Accept        : "application/json"

Get list of records

GET {url}/api/leads
In result You will get list of paginated records

Get single record

GET {url}/api/leads/get/{id}
In result You will get single record

Create new record

POST {url}/api/leads/create
In result, You will get a single created record

Update record

POST {url}/api/leads/update/{id}
In result, You will get a single updated record

Delete record

DELETE {url}/api/leads/delete/{id}
In result, You will get information if record was deleted

 

Creating new rest API

Create new Controller that extends CrudApiController

<?php

namespace Modules\Api\Http\Controllers;

use Modules\Api\Http\Controllers\Base\CrudApiController;
use Modules\Api\Http\Requests\LeadApiRequest;
use Modules\Leads\Entities\Lead;

class LeadsApiController extends CrudApiController
{

    // Entity Class
    protected $entityClass = Lead::class;

    // Module name
    protected $moduleName = 'leads';

    // Language pattern
    protected $languageFile = 'leads::leads';

    // What relations to get with single entity
    protected $with = [
      'leadStatus'
    ];

    // Permissions (same as in Web Crud Controller)
    protected $permissions = [
        'browse' => 'leads.browse',
        'create' => 'leads.create',
        'update' => 'leads.update',
        'destroy' => 'leads.destroy'
    ];

    // Needed for notifications
    protected $showRoute = 'leads.leads.show';

    protected $storeRequest = LeadApiRequest::class;

    protected $updateRequest = LeadApiRequest::class;

    public function __construct()
    {
        parent::__construct();
    }

}

Create request file

<?php
namespace Modules\Api\Http\Requests;

use App\Http\Requests\Request;

/**
 * Lead API Request
 *
 * Class LeadApiRequest
 * @package Modules\Api\Http\Requests
 */
class LeadApiRequest extends Request
{

    /**
     * Determine if the user is authorized to make this request.
     *
     * @return bool
     */
    public function authorize()
    {
        return true;
    }

    public function rules()
    {
        return [
            'first_name' => 'required',
            'last_name' => 'required',
            'email' => 'sometimes|nullable|email',
            'secondary_email' => 'sometimes|nullable|email',
            'annual_revenue' => 'sometimes|nullable|numeric',
            'website' => 'sometimes|nullable|url',
            'no_of_employees' => 'sometimes|nullable|numeric'
        ];
    }
}

Add rest api route

Route::group(['prefix' => 'tickets'], function (){
        Route::get('/','TicketsApiController@index');
        Route::get('get/{id}','TicketsApiController@get');
        Route::post('create','TicketsApiController@store');
        Route::post('update/{id}','TicketsApiController@update');
        Route::delete('delete/{id}','TicketsApiController@destroy');
    });

How to extend with custom methods?

For best separation of generic and custom code create new controller in /Modules/Api/Http/Controllers/ folder , add routes, requests .. and don’t forget to add middleware(‘jwt:auth’);
